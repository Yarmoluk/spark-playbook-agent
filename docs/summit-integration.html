🔗 SPARK Agent Summit Integration

Technical Specification for Summit Platform Team

API Contract • Data Models • Integration Patterns • Deployment

📋 Overview

This spec defines how the SPARK playbook agent integrates with Slalom's Summit knowledge platform to:

  • Search and retrieve playbook templates, methodologies, accelerators
  • Index engagement context and constraint parameters
  • Query patterns and delivery models with trade-off metadata
  • Store generated playbooks and source attribution logs
  • Expose agent status and output bundle via REST API

🏗️ System Architecture

Summit Layer

Asset indexing, search API, metadata store, version control

Claude Agent Layer

Orchestration, synthesis, composition, quality gates

Storage Layer

Playbook bundles, patterns, source logs, template library

API Layer

Trigger endpoints, status polling, output retrieval, webhooks

🔌 Core API Contracts

1. Trigger Playbook Generation

POST /v1/spark/playbooks/generate

Trigger agent to assemble a new playbook from engagement context.

Request Body

{ "engagement_id": "ENG-2026-001", "client_name": "Canadian Agency XYZ", "vertical": "cloud-transformation", "scope": { "services": ["migration", "modernization", "training"], "systems": 8, "users": 250, "locations": 3 }, "constraints": { "timeline_months": 12, "budget_usd": 500000, "risk_tolerance": "medium", "regulatory": ["PIPEDA", "SOC2"] }, "stakeholders": [ { "name": "John Sponsor", "role": "CIO", "email": "john@agency.ca" } ], "tags": ["government", "cloud", "legacy-migration"], "preferred_patterns": ["phased", "agile"], "exclude_assets": ["internal_pricing"] }

Request Parameters

Field Type Required Description
engagement_id string Yes Unique engagement identifier for tracking & audit
client_name string Yes Client organization name
vertical enum Yes cloud-transformation | data-modernization | ai-adoption | security | other
scope object Yes Engagement scope details (services, scale, geography)
constraints object Yes Timeline, budget, risk, compliance requirements
stakeholders array No Key contacts for approval & sign-off
tags array No Keywords for asset search filtering
preferred_patterns array No Preferred delivery models to prioritize
exclude_assets array No Asset tags to exclude (e.g., internal pricing)

3. Retrieve Output Bundle

GET /v1/spark/jobs/{job_id}/artifacts/{artifact_type}

Download specific artifact from completed job.

Supported Artifact Types

Type Format Description
playbook HTML, DOCX Client-ready playbook with all sections
pattern_comparison JSON, HTML Trade-off analysis matrix
alignment_deck PPTX, PDF Sponsor presentation
source_attribution JSON Complete source log with links
quality_checklist JSON 10-point quality gate results

🔄 Integration Workflow

Summit Search & Asset Retrieval

1. Agent Receives Engagement Context
Job triggered with client vertical, scope, constraints, tags
2. Execute Parallel Searches
Summit API queries:
• GET /v1/assets/search?type=methodology&tag=cloud-transformation
• GET /v1/assets/search?type=template&tag=phased-delivery
• GET /v1/patterns?vertical=cloud-transformation&complexity=medium
3. Index Results with Metadata
Store asset IDs, URLs, summaries, source info in agent memory
4. Constraint Matching
Score each pattern against scope, timeline, budget, risk tolerance
5. Synthesize Playbook
Assemble playbook from selected pattern + sourced content
6. Store Output Bundle
Save playbook, comparison matrix, alignment deck, source log to Summit

POST /v1/assets/index

🧪 Integration Testing Checklist

  • Summit API connectivity test (search, retrieve, index)
  • Pattern scoring algorithm validation (constraint matching)
  • Playbook assembly completeness (all sections populated)
  • Source attribution accuracy (100% of content has source)
  • Quality gate execution (10 checks pass/fail correctly)
  • Output bundle generation (all 6 artifacts created)
  • Concurrent job handling (multiple jobs simultaneously)
  • Error handling (graceful fallbacks, clear error messages)
  • Performance benchmark (2–6 hour SLA validation)
  • Load testing (10+ simultaneous jobs)

⚠️ Implementation Notes

Asset Indexing: Summit team must establish asset taxonomy (type, tags, verticals) before agent launch. Recommend starting with 20–30 core cloud-transformation assets.
Pattern Library: Build pattern models in Summit as queryable objects, not static docs. Include suitability scores for rapid constraint matching.
Security: Mark sensitive assets (internal pricing, IP) with reusability: restricted. Agent respects exclusion lists passed in trigger request.
Versioning: Track asset versions in Summit. Playbook generation should capture asset version IDs in source attribution for auditability.
Rate Limiting: Set Summit API rate limits for agent searches (suggest 100 RPS burst, 50 RPS sustained). Agent job queue prevents overload.
